# 002. Date Time Timezone Policy

## 상태

Accepted

## 배경

BaseKit은 여러 사용자, 업체, 고객사, 업무 단위가 같은 데이터를 조회하고 처리할 수 있는 업무시스템 참조 아키텍처입니다. 날짜와 시간은 저장 기준, 화면 표시 기준, 업무 판정 기준을 명확히 분리해야 합니다.

특히 입찰 마감, 신청 마감, 승인 시각, 로그인 기록처럼 시간 기준이 중요한 업무에서는 화면에 표시되는 현지 시간이 다르더라도 실제 업무 판정 시점은 하나로 유지되어야 합니다.

## 결정

BaseKit의 `*_DT` / `*_AT` 컬럼은 시스템 기준 타임존으로 저장합니다.

기본 시스템 기준 타임존은 `Asia/Seoul`입니다.

```text
SYSTEM_TIMEZONE_ID = Asia/Seoul
```

향후 시스템 설정값인 `SYSTEM_TIMEZONE_ID`를 통해 기준 타임존을 변경할 수 있게 설계합니다. 저장, 정렬, 비교, 마감 여부 판단은 이 시스템 기준 시간 값을 기준으로 처리합니다.

대상 컬럼 예시는 다음과 같습니다.

```text
CREATED_AT
UPDATED_AT
LAST_LOGIN_AT
ACCOUNT_EXPIRED_AT
PASSWORD_CHANGED_AT
LOGIN_AT
LOGOUT_AT
BID_CLOSE_DT
SUBMIT_DEADLINE_DT
```

## 화면 표시 기준

화면 표시 시에는 로그인 사용자의 `TIMEZONE_ID`로 변환합니다.

사용자가 속한 업체 또는 고객사의 업무 화면에서는 사용자 `TIMEZONE_ID`뿐 아니라 업체, 고객사, 업무 기준 `TIMEZONE_ID`를 표시 기준으로 사용할 수 있습니다. 단, 표시 기준이 달라도 저장된 기준 시간과 업무 판정 기준은 바뀌지 않습니다.

## 정렬, 비교, 마감 판정

정렬, 비교, 마감 여부 판단은 저장된 기준 시간으로 처리합니다.

화면에 표시된 현지 시간을 다시 기준값으로 사용하지 않습니다. 화면 시간은 사용자 이해를 위한 표시값이고, 실제 업무 로직은 시스템 기준 타임존으로 저장된 값에 대해 수행합니다.

## 입찰 및 마감 시간

입찰 마감 같은 업무 시간은 하나의 기준 시간으로 저장합니다.

모든 참여자에게 실제 마감 순간은 동일해야 합니다. 업체별 화면에서는 각 사용자 또는 업체의 `TIMEZONE_ID` 기준으로 시간이 다르게 표시될 수 있지만, 실제 마감 순간은 하나입니다.

예를 들어 시스템 기준 타임존이 `Asia/Seoul`이고 입찰 마감 시간이 다음과 같이 저장되었다고 가정합니다.

```text
저장 기준: Asia/Seoul
마감 시간: 2026-06-14 15:00
```

화면 표시는 사용자 또는 업체 기준 타임존에 따라 다르게 보일 수 있습니다.

```text
Asia/Seoul: 2026-06-14 15:00
UTC: 2026-06-14 06:00
America/Los_Angeles: 2026-06-13 23:00
```

표시 시간은 다를 수 있지만 실제 마감 순간은 동일합니다.

## 적용 원칙

- 저장: 시스템 기준 타임존
- 기본 시스템 기준 타임존: `Asia/Seoul`
- 설정 확장: `SYSTEM_TIMEZONE_ID`
- 표시: 로그인 사용자 또는 업무 기준 `TIMEZONE_ID`
- 정렬: 저장된 기준 시간
- 비교: 저장된 기준 시간
- 마감 판정: 저장된 기준 시간

## 기대 효과

- 시스템 전체의 저장 기준 시간이 일관됩니다.
- 한국 업무 기준 시스템을 기본값으로 삼으면서도 향후 기준 타임존 변경 여지를 남깁니다.
- 사용자, 업체, 고객사별 화면 표시 요구를 처리할 수 있습니다.
- 화면 표시 시간과 실제 업무 판정 시간을 분리하여 마감 판정 오류를 줄입니다.

## 주의사항

화면에 표시된 시간만으로 정렬, 비교, 마감 판정을 수행하지 않습니다.

외부 연동이나 국제 표준 시각이 필요한 API에서는 시스템 기준 시간과 함께 기준 타임존 정보를 명확히 전달해야 합니다.
